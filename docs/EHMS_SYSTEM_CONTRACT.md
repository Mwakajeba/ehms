## 2. PURPOSE

The Provider grants the Client a license to use the **Enterprise Hospital Management System (EHMS)** — a web-based integrated platform for hospital operations, inventory control, multi-branch administration, user access control, and clinical workflows.

This document describes the **licensed modules and features** included under this Agreement.

---

## 3. SYSTEM OVERVIEW

EHMS is a multi-company, multi-branch system that supports:

- Centralized data per organization (company)
- Branch-level operations with user branch assignment
- Role-based access control and audit logging
- Integration between hospital services, billing, and inventory

---

## 4. LICENSED MODULES & FEATURES

### 4.1 Hospital Management

End-to-end clinical and front-office workflows for hospitals and clinics.

| Area | Features Included |
|------|-------------------|
| **Reception** | Patient registration with auto-generated MRN; patient search, view, and edit; insurance type assignment; visit creation with selectable visiting date (backdating); department routing; pre-billing via sales invoices; welcome SMS; patient deletion approval workflow |
| **Cashier** | Visit bill viewing; payment processing; bill clearance; waiting lists |
| **Triage** | Vital signs recording; patient routing to departments; queue management |
| **Doctor** | Consultations; diagnosis and treatment plans; prescriptions; ordering lab, ultrasound, audiology, dental, and pharmacy bills; viewing results |
| **Laboratory (Lab)** | Test queue (waiting / in service); results entry; result status (pending / ready); printing |
| **Ultrasound** | Examination queue; results entry and management; printing |
| **Audiology** | Hearing assessment queue; results entry and management; printing |
| **Dental** | Dental procedures and billing workflow |
| **Pharmacy** | Dispensing workflow; pharmacy bills linked to visits |
| **RCH, Vaccination, Injection, Family Planning** | Specialized department workflows as configured |
| **Insurance** | Configurable insurance types per company; receivable chart account per insurer; invoice payment by insurance (insurance receivable / trade receivable GL posting) |
| **Sales integration** | Pre-billing and service bills as sales invoices; payment methods: bank/cash, customer deposits, insurance; link from invoice to hospital visit/patient |
| **Administration** | Hospital departments setup; insurance types (Settings); operational dashboard with pending counts per module |

---

### 4.2 Inventory Management

Stock and item control for medicines, consumables, equipment, and services used across the organization.

| Area | Features Included |
|------|-------------------|
| **Items & categories** | Product and service items; categories; item codes; units of measure; active/inactive status |
| **Branch pricing** | Default selling price per item; per-branch price overrides |
| **Stock movements** | Receipts, issues, adjustments; movement history per item and location |
| **Transfers** | Inter-location / inter-branch stock transfers; transfer requests with approve/reject |
| **Write-offs** | Stock write-off recording and tracking |
| **Stock reports** | Stock levels by location; stock reports and location views |
| **Physical counts** | Count periods and sessions; freeze, count, variance review; approve/reject count sessions |
| **Opening balances** | Inventory opening balance setup |
| **Import** | Bulk item import via template |
| **Settings** | Inventory accounts linkage; cost method configuration; inventory locations per branch |
| **Integration** | Items used on hospital pre-bills, pharmacy, and sales invoices |

---

### 4.3 Branch Management

Operate multiple sites under one company with controlled access.

| Area | Features Included |
|------|-------------------|
| **Branches** | Create, edit, and deactivate branches; branch details and settings |
| **Locations** | Inventory/storage locations linked to branches |
| **User assignment** | Assign users to one or more branches |
| **Session context** | Users select active branch (change branch); operations scoped to current branch |
| **Data scope** | Patients, visits, stock, and transactions filtered by company and branch |
| **Per-branch inventory** | Stock and pricing managed per branch where applicable |

---

### 4.4 User Management

Secure access and administration of system users.

| Area | Features Included |
|------|-------------------|
| **Users** | Create, edit, activate/deactivate users; user profiles and passwords |
| **Roles** | Role definitions with grouped permissions |
| **Permissions** | Granular permission control per module and action |
| **Role assignment** | Assign multiple roles per user |
| **Branch assignment** | Restrict user access to assigned branches |
| **Menu access** | Role-based menu visibility configuration |
| **Company scope** | Users operate within their assigned company |
| **Audit** | Activity logging on key records (where enabled) |

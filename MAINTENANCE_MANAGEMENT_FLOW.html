1. System Overview

The Maintenance Management System is a comprehensive module within the SmartAccounting Assets Management system that handles the complete lifecycle of asset maintenance from request initiation to cost classification and accounting integration.

Key Features

  • Preventive Maintenance: Scheduled maintenance based on time or usage
  • Corrective Maintenance: Breakdown and repair requests
  • Major Overhaul: Significant improvements and capital upgrades
  • Cost Tracking: Material, labor, and other direct costs
  • Automatic Classification: Expense vs. Capitalization logic
  • GL Integration: Automatic journal entry posting
  • Asset Updates: Automatic cost and depreciation adjustments

2. Pre-Setup Configuration

2.1 Maintenance Types Setup

Location: /asset-management/maintenance/types

Purpose: Define maintenance categories that determine expense/capitalization logic.

Steps:

  1. Navigate to Maintenance → Types
  2. Click "Create New Maintenance Type"
  3. Fill in the form:
    • Code: Unique identifier (e.g., "PM-001")
    • Name: Descriptive name (e.g., "Preventive Maintenance - Monthly")
    • Type: Select from:
      • preventive: Routine scheduled maintenance
      • corrective: Breakdown repairs
      • major_overhaul: Significant improvements
    • Description: Detailed explanation
    • Status: Active/Inactive
  4. Save

2.2 Maintenance Settings Configuration

Location: /asset-management/maintenance/settings

A. GL Accounts

  • Maintenance Expense Account: Default P&L account for routine maintenance expenses
  • Maintenance WIP Account: Work-in-Progress account for costs during execution
  • Asset Capitalization Account: GL account for capitalized maintenance costs

B. Capitalization Thresholds

  • Capitalization Threshold Amount: Minimum cost (TZS) to qualify for capitalization
    • Default: TZS 2,000,000
    • Example: Any maintenance costing more than 2M TZS may be capitalized
  • Capitalization Life Extension Threshold: Minimum life extension (months) to qualify
    • Default: 12 months
    • Example: If maintenance extends asset life by 12+ months, it may be capitalized

3. Maintenance Request Flow

Step 1: Request Initiation

Location: /asset-management/maintenance/requests/create

Triggering Events:

  • Preventive: Scheduled maintenance based on calendar or usage
  • Corrective: Asset breakdown or malfunction
  • Planned Improvement: Upgrades or enhancements

Process:

  1. Select Asset
    • System auto-fetches: Asset ID, name, location, custodian, cost center, department
  2. Select Maintenance Type
    • Choose from configured maintenance types
    • System determines default classification logic
  3. Enter Request Details:
    • Trigger Type: Preventive, Corrective, or Planned Improvement
    • Priority: Low, Medium, High, Urgent
    • Description: What needs to be done
    • Issue Details: Problem description (for corrective)
    • Requested Date: When maintenance is needed
    • Preferred Start Date: Preferred start date
    • Attachments: Upload relevant documents/photos
  4. Submit Request
    • System generates unique Request Number (e.g., "MR-2025-001")
    • Status: pending
    • Notification sent to supervisor

Step 2: Supervisor Approval

Location: /asset-management/maintenance/requests/{id}

Process:

  1. Supervisor Reviews Request
    • Views asset details
    • Reviews issue description
    • Checks attachments
    • Verifies necessity
  2. Decision Options:
    • Approve: Status → approved, ready for Work Order creation
    • Reject: Status → rejected, request closed
    • Request More Information: Add notes, remains pending

4. Work Order Creation and Execution

Step 3: Work Order Creation

Location: /asset-management/maintenance/work-orders/create

Process:

  1. Link to Maintenance Request (if applicable)
    • Select approved maintenance request
    • System auto-populates: Asset details, Maintenance type, Request details
  2. Work Order Details:
    • WO Number: Auto-generated (e.g., "WO-2025-001")
    • Asset: Selected from approved request
    • Maintenance Type: From request
    • Execution Type:
      • in_house: Internal technicians
      • external_vendor: External contractor
      • mixed: Combination
    • Vendor/Technician: Select supplier or internal user
    • Estimated Dates: Start date, Completion date
    • Estimated Costs: Labor, Material, Other costs
    • Estimated Downtime: Hours asset will be unavailable
    • Budget Reference: Link to budget if applicable
    • Work Description: Detailed work plan
  3. Submit Work Order
    • Status: draft
    • Requires approval before execution

Step 4: Work Order Approval

Location: /asset-management/maintenance/work-orders/{id}/approve

Process:

  1. Review Work Order
    • Verify estimates
    • Check budget availability
    • Confirm vendor/technician assignment
  2. Approve
    • Status: approved
    • Notifications sent to: Assigned technician/vendor, Asset custodian, Department head

Step 5: Work Order Execution

Location: /asset-management/maintenance/work-orders/{id}/execute

Process:

  1. Start Work
    • Update status to in_progress
    • Record actual_start_date
    • Begin cost capture
  2. Cost Capture During Execution
    • Material Costs: Select inventory items, enter quantity and unit cost
    • Labor Costs: Internal (technician hours/rate) or External (vendor invoice)
    • Other Direct Costs: Fuel, subcontracting, logistics
  3. Track Progress
    • Update work performed notes
    • Record actual downtime hours
    • Add technician notes
    • Upload completion photos

Step 6: Work Order Completion

Location: /asset-management/maintenance/work-orders/{id}/complete

Process:

  1. Technician Marks Complete
    • Update status to completed
    • Record actual_completion_date
    • Enter work_performed details
    • Finalize all costs
  2. System Actions
    • Calculate total actual costs
    • Update completed_by and completed_at
    • Status: completed
    • Cost Classification: pending_review
  3. Notification Sent
    • Finance/Asset Accountant for review
    • Asset custodian
    • Department head

5. Cost Capture and Recording

Cost Types

1. Material Costs

Source: Inventory items

Process:

  1. Navigate to Work Order → Execute → Add Cost
  2. Select cost type: material
  3. Select inventory item
  4. Enter quantity and unit cost
  5. System calculates total cost
  6. Optional: Link to Purchase Order or Invoice

2. Labor Costs

Source: Internal technicians or external vendors

Internal Labor:

  1. Select cost type: labor
  2. Select technician (user)
  3. Enter hours worked
  4. System applies hourly rate (from HR/Payroll if integrated)
  5. Calculate: hours × rate = labor cost

External Labor:

  1. Select cost type: labor
  2. Select supplier/vendor
  3. Link to Purchase Invoice
  4. System pulls amount from invoice

3. Other Direct Costs

Source: Various (fuel, subcontracting, logistics, etc.)

  1. Select cost type: other
  2. Enter description
  3. Enter amount
  4. Optional: Link to supplier invoice

6. Cost Classification Logic

Step 7: Finance Review and Classification

Location: /asset-management/maintenance/work-orders/{id}/review

Purpose: Determine if maintenance costs should be expensed or capitalized.

Classification Rules

A. Routine Maintenance (Expense)

Criteria:

  • Cost < Capitalization Threshold Amount (default: 2M TZS)
  • Life extension < Capitalization Life Extension Threshold (default: 12 months)
  • Standard preventive or corrective maintenance
  • No significant improvement to asset

Classification: expense

Accounting Treatment:

Dr. Maintenance Expense (P&L)
Cr. Maintenance WIP

Asset Impact:

  • Asset cost: No change
  • Depreciation: No change
  • Asset history: Recorded as expense

B. Capital Improvement/Major Overhaul (Capitalized)

Criteria (any of the following):

  • Cost ≥ Capitalization Threshold Amount
  • Life extension ≥ Capitalization Life Extension Threshold
  • Significant improvement to asset functionality
  • Major overhaul that extends useful life

Classification: capitalized

Accounting Treatment:

Dr. Fixed Asset (Improvement)
Cr. Maintenance WIP

Asset Impact:

  • Asset cost: Increased by capitalized amount
  • Depreciation: Recalculated based on new cost
  • Useful life: Extended (if applicable)
  • TRA Tax Book: Updated (if applicable)

7. GL Posting and Asset Updates

Step 8: GL Posting

Location: Automatic after classification (or manual trigger)

A. Expense Classification

Journal Entry:

Account Debit Credit
Maintenance Expense XXX -
Maintenance WIP - XXX

B. Capitalized Classification

Journal Entry:

Account Debit Credit
Fixed Asset (Improvement) XXX -
Maintenance WIP - XXX

Asset Updates:

  1. Update Asset Cost: $asset->cost += $capitalizedAmount;
  2. Recalculate Depreciation: New depreciation basis = Original cost + Capitalized amount
  3. Update Useful Life (if extended): $asset->useful_life_months += $lifeExtensionMonths;
  4. TRA Tax Book Update (if applicable): Update tax book value and adjust tax depreciation

Step 9: Maintenance History Creation

Location: Automatic after GL posting

Process:

  1. Create History Record
    • Table: maintenance_history
    • Links to: Asset, Work Order, Maintenance Type
    • Records: Completion date, Total cost, Cost classification, Life extension, Vendor/Technician, Work performed, Notes
  2. Update Asset History
    • Last maintenance date
    • Last maintenance cost
    • Maintenance frequency tracking
    • Next maintenance schedule (for preventive)

8. Reports and Analytics

Dashboard KPIs

Location: /asset-management/maintenance

Metrics:

  • Total Requests: All maintenance requests
  • Pending Requests: Awaiting approval
  • Open Work Orders: Approved, In Progress, On Hold
  • Completed This Month: Completed work orders
  • Total Cost YTD: Year-to-date maintenance costs
  • Capitalized vs. Expensed: Percentage breakdown
  • Average Cost per Asset: Cost efficiency metric
  • Downtime Hours: Total asset downtime

Operational Reports

  • Upcoming Maintenance Schedule: Preventive maintenance due, sorted by due date
  • Open Work Orders: Status: Approved, In Progress, On Hold, with filters
  • Downtime Report: Asset downtime by period, impact on operations, trend analysis
  • Vendor Performance: Vendor response time, cost comparison, quality ratings

Financial Reports

  • Cost per Asset/Department: Total maintenance cost by asset, cost by department/cost center, budget vs. actual
  • Capitalized vs. Expensed: Breakdown by classification, trend over time, impact on P&L and Balance Sheet
  • Life Extension Analysis: Assets with extended useful life, capitalized amounts, ROI on capital improvements
  • Budget vs. Actual: Maintenance budget performance, variance analysis, forecast vs. actual
  • Depreciation Forecast: Impact of capitalized maintenance, revised depreciation schedules, tax implications

10. User Roles and Permissions

Required Permissions

  • Maintenance Request Creator: create maintenance requests, view maintenance requests
  • Supervisor: approve maintenance requests, view maintenance requests, view work orders
  • Maintenance Technician: view work orders, execute work orders, add work order costs, complete work orders
  • Finance/Asset Accountant: review work orders, classify work orders, post to GL, view maintenance reports
  • Maintenance Administrator: All permissions, manage maintenance types, manage maintenance settings, delete maintenance records

Workflow Summary

Maintenance Request → Supervisor Approval → Work Order Creation → Work Order Approval → Execution & Cost Capture → Completion → Finance Review & Classification → GL Posting → Asset Updates → Maintenance History

Key Database Tables

  • maintenance_types: Maintenance type definitions
  • maintenance_requests: Maintenance request records
  • asset_maintenance_work_orders: Work order records
  • asset_maintenance_work_order_costs: Individual cost records
  • maintenance_history: Completed maintenance history
  • maintenance_settings: System configuration

Best Practices

  1. Preventive Maintenance: Schedule regular maintenance to prevent breakdowns, track maintenance intervals, update schedules based on actual usage
  2. Cost Tracking: Capture all costs in real-time, link to source documents (POs, invoices), maintain audit trail
  3. Classification: Review all completed work orders, apply capitalization rules consistently, document classification decisions
  4. Asset Management: Update asset costs promptly, recalculate depreciation after capitalization, maintain accurate asset history
  5. Reporting: Review maintenance costs regularly, analyze trends and patterns, optimize maintenance strategies
